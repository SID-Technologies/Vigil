# Vigil — Design Context

This file persists design intent across sessions. Read it before making any
visual / copy / accessibility decision. Update it when intent shifts (don't
let it rot).

## Design Context

### Users

Vigil's users are people whose ISP or property manager has told them
"everything's fine" when their video calls keep dropping. They need
evidence to push back, and they don't want to become network engineers
to get it. The audience is technical-leaning but not all developers —
think frustrated remote workers, friends with bad home Wi-Fi, family
members on the wrong side of a managed-property contract.

The job: produce credible reliability evidence (uptime %, outage
timestamps, RTT distributions) and share it without effort. Vigil should
make the proof feel effortless to generate and dignified to hand over.

Emotional target: **confidence** (this is real data, not vibes),
**calm** (the watchman watches so they don't have to), and a quiet
**vindication** when an outage finally gets caught and timestamped.

### Brand Personality

Three words: **vigilant, dignified, friendly.**

- **Vigilant** ≠ paranoid. Vigil watches without alarming.
- **Dignified** ≠ stiff. Engineering-grade visuals; the data is the hero.
- **Friendly** ≠ cute. Approachable copy, no mascots, no gamification,
  no condescension. Sounds like a watchman reporting in, not an alarm.

Voice examples:
- ✅ "All systems nominal" / "Network outage in progress" / "Quiet night
   so far. Nothing's broken."
- ✅ "Click a tile to compare it on the chart"
- ❌ "Yikes! Something's wrong!!" (too panicked)
- ❌ "Selectable items" (too terse)
- ❌ "Awesome! 🎉 You earned 30 days of uptime!" (gamified / cute)

### Aesthetic Direction

**Theme**: Night Watch is the canonical Vigil aesthetic — slate
`#0b1116` dark with watchfire amber `#e0a458` accent, Space Grotesk
typography. Five theme styles ship (default / torch / retro / odyssey /
nightwatch); users can pick. Dark mode is the default and brand-defining
view (a watchman works at night). Light mode is parity-quality but
secondary in priority.

**References** (in priority order):
- **GitHub** — data tables, dense layouts, clarity over decoration
- **Vercel** — whitespace, dark-mode-first, restrained UI
- **Linear** — selection-mode header transformations, keyboard-driven
  workflows, motion that signals state changes

**Anti-references** (don't drift toward these):
- Gradient meshes, floating orbs, "AI-generated dashboard" glow
- Skeuomorphic dials, analog gauges, photographic textures
- Cute mascots, animated celebrations, confetti
- Pill-shaped-everything (we use $1.5–$3 border radii, not 999 globally)
- Heavy drop shadows, depth abuse, glassmorphism for its own sake

### Design Principles

1. **Evidence first.** Every screen exists to produce or read evidence —
   uptime, timestamps, error codes, RTT percentiles. Decoration earns
   its space only when it helps the user *see* the evidence faster.
2. **Live but not loud.** Subtle pulses (2.5s watchfire dot, refresh
   tickers) convey aliveness. Never blink, flash, or animate
   aggressively. Always respect `prefers-reduced-motion`.
3. **Dense but legible.** Borrow GitHub's spreadsheet density. Fixed
   column widths, tabular numerals, hierarchy through scale and color
   rather than padding.
4. **Color earns meaning.** Watchfire amber = "current value" / primary
   action. Per-target palette = "this exact target's data" (deterministic
   so the same target is the same color across pages). Red = "actually
   wrong." Don't decorate with color; reserve it for signal.
5. **Voice like a watchman, not an alarm.** Reports state matter-of-
   factly with a hint of warmth. Approachable enough that a friend
   without a CS degree can read a report and feel informed. Never panics,
   never gamifies, never condescends.

### Accessibility (target: friend-and-family ready)

- **Contrast**: body text ≥ 4.5:1, UI / large text ≥ 3:1. Verified
  manually on the Night Watch palette; check before approving new tones.
- **Color is never the sole signal.** Pair status colors with icons or
  text — `failing` tile gets red border + a red dot + the error code,
  not just "more red."
- **Color-blind safety**: red and green should not be the only
  differentiator. Where success/failure share screen real estate,
  include shape/icon variation. The per-target palette deliberately
  avoids placing red and green next to each other in hash-index space.
- **Keyboard navigation**: every interactive element reachable via Tab.
  Visual focus rings on all focusable elements. Modal dismissal via Esc.
- **Motion**: respect `prefers-reduced-motion`. The watchfire pulse,
  chart entry animations, hover scales — all should fall back to opacity
  fades or disable when the OS preference is set.
- **No flashing > 3 times per second** (WCAG seizure guideline).
- **Tooltip readability**: chart tooltips show all selected series at
  the hovered timestamp, sorted descending by value, with both color
  swatch *and* text label so colorblind users can still parse the
  highest-RTT line.
